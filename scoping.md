# Project Scoping

## 1. Data Migration

In order to export data out of WordPress I would consider 2 options, depending on the tools available, budget and time constraints:

1. Native Solution - either using WordPress Tolls or direct database access to export the data rows (products, customers and orders) in a CSV file and format the columns and data.
2. Plugin Solution - with a paid plugin like [WP All Import](https://www.wpallimport.com/export-wordpress/) - I've used it in the past and it lets you format the spreadsheet structure directly and execute functions to convert data. 

The quote depends on the quality of data WordPress has and how organised it is but the average would be *1.5 hours*.

---

The import to Shopify, after having formatted the data, would be fairly simple using Shopify native tools or the [Matrixify app](https://matrixify.app/) if required, taking about *30 minutes*.

I would then, spend some time in QA to ensure that the data migrated is correct, taking another *30 minutes*. 

**Est. total: 2.5 hours**


## 2. Templates Development

Usually, my way of quoting brochure templates is composed of 3 steps:

1. Quoting individual sections/blocks;
2. Quoting global components;
3. Allowing some time for reviews/feedback (15-20%).


### 2.1. Static Templates

Given that the homepage has 4 standard sections, I would assume that the other 4 static pages would follow a similar structure. This way, I am viewing these static templates summing up to around 16-20 sections or maybe less, since some of them might get reused.

My estimates to only customise a standard section would be under *1 hour* and to build one from scratch would vary between *2-4 hours*, depending on complexity.

**Est. total: 45 hours**


### 2.2. Homepage Sections

Since all sections on the homepage are pretty standard and already included in the **Horizon theme**, this would be my theme of choice to use on this project.

Regarding the quotes for the sections, I believe I would spend at most *2 hours* on each, pending designs complexity.

**Est. total: 7 hours**


## 3. Product Template

For the product template, the add-to-cart component is already pretty similar to *Horizon's* default product template, meaning that development time would focus on building the image grid, color swatches and info and both the related-products and image-text sections.

- The colour swatches feature, using sibling products instead of variants would take roughly *45 minutes*;
- The development of the USP icons and accordions would take about *1 hour*;
- You may like section would likely reuse another Horizon section, taking about *45 minutes* to customise;
- Image with text section could be fully reused from Horizon's XXXXXX section, taking *20 minutes* to customise;
- Global CTA section would be around *1 hour*;

**Est. total: 4 hours**


## 4. Collection Template

The collection template can be the most volatile template depending on filters requirements, and display options. Generally a re-theme would take me between 1-2 hours, whereas additional features could increase that time significantly.

**Est. total: 2 hours**


## 5. Custom Features

### 5.1. Mega Menu

I generally tend to custom code all mega-menus, trying to achieve the most fluid and accessible results for both desktop/mobile users. 

**Est. total: 5 hours**


### 5.2. Collection Filters

For complex filter solutions I would generally rely on an app rather than custom coding it. One of my choices would be [Smart Product Filter & Search](https://apps.shopify.com/product-filter-and-search) depending on task requirements.

**Est. total: ~3 hours**


### 5.3. Shipping Progress Bar

I haven't built anything similar, but I believe my approach to this would be using an app to save development time. If the budget was enough and there were APIs to connect with the shipping courier I would also be able to custom code it retrieving data directly for the courier API.

**Est. total: ~5 hours**


### 5.4. Back in Stock

Some of these third party marketing tools with Shopify App integrations, already come with Back in Stock features out of the box, so the setup here is pretty much plug n' play.

Klaviyo, specifically, has Back In Stock forms, that will notify the customer. I've implemented similar features to WordPress stores before and the setup is simple.

**Est. total: ~1.5 hours**


### 5.5. Instagram Feed

Again, only relying on an app for this feature and the setup should be mostly plug n' play with small customisations to match the store.

One app I've used in the past is [Instafeed](https://apps.shopify.com/instafeed).

**Est. total: ~1 hour**
